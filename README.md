# Обнаружение аномалий в вибрационных сигналах подшипников качения

## Описание

Данный репозиторий содержит исходный код выпускной квалификационной работы (бакалавра) на тему: **"Разработка ML-модели машинного обучения для обнаружения аномалий в работе промышленного оборудования"**

## Датасеты

В работе использованы два публичных набора данных:

| Датасет | Описание | Источник |
|---------|----------|----------|
| **CWRU** | Вибрационные сигналы подшипников с искусственными дефектами | [Case Western Reserve University](https://engineering.case.edu/bearingdatacenter/welcome) |
| **IMS** | Процесс деградации подшипника до отказа | [NASA Ames Research Center](https://data.nasa.gov/dataset/ims-bearings) |

> **Примечание:** Исходные файлы датасетов не включены в репозиторий из-за большого объёма. Перед запуском ноутбуков необходимо скачать датасеты и поместить их в соответствующие директории согласно инструкциям в `1_Preprocessing.ipynb`.

## Модели

Обучены и сравнены следующие модели:
- **Isolation Forest** - алгоритм обнаружения аномалий на основе случайных разбиений.
- **One-Class SVM** - метод опорных векторов для одноклассовой классификации.
- **Autoencoder** - полносвязная нейронная сеть для реконструкции признаков.
- **LSTM-Autoencoder** - рекуррентная нейронная сеть для анализа временных рядов.

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/OnufrienkoVS/bearing-anomaly-detection.git
cd bearing-anomaly-detection
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Запуск веб-интерфейса

```bash
cd prototype
streamlit run app.py
```

После запуска веб-интерфейс будет доступен по адресу: http://localhost:8501

Функциональность веб-интерфейса:

- Загрузка файлов с вибрационными сигналами (форматы TXT, NPY);
- Подключение к OPC UA серверу для мониторинга в реальном времени;
- Визуализация ошибки реконструкции;
- Отображение обнаруженных аномалий;
- История обработки с возможностью экспорта в CSV.

### 4. Запуск тестового OPC UA сервера

Для тестирования системы без подключения к реальному оборудованию запустите симулятор:
```bash
cd prototype
python opc_server.py --file <file_name> --repeat <n>
```

Параметры:

- `--file` - путь к файлу с вибрационным сигналом (например, test_data/normal.txt).
- `--repeat` - количество повторений файла (сколько раз сигнал будет передан в цикле).
